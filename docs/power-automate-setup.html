TrailerPark - Power Automate Setup Guide

One-time setup to automatically save incoming emails to OneDrive for the TrailerPark app.

What This Does

This sets up an automatic flow that:

  1. Watches for new emails arriving in your Outlook inbox
  2. Saves each email's details (sender, subject, body) as a metadata.json file in OneDrive
  3. Saves any email attachments (PDFs, Excel files, images) alongside it
  4. Skips signature images (logos, social icons) automatically
  5. OneDrive syncs everything to your local computer, where the TrailerPark app picks it up
No extra cost: All connectors used are standard and included with your M365 Business plan. No premium Power Automate license is needed.

Prerequisites

  • Access to make.powerautomate.com with your M365 account
  • OneDrive desktop sync app running on the office computer

Setup Steps

Step 1: Create a New Flow

  1. Go to make.powerautomate.com and sign in
  2. Click My flows in the left sidebar
  3. Click + New flow → Automated cloud flow
  4. Name it: TrailerPark - Save Emails to OneDrive
  5. In "Choose your flow's trigger", search for and select "When a new email arrives (V3)" (Office 365 Outlook)
  6. Click Create

Step 2: Configure the Email Trigger

In the trigger settings that appear:

  1. Folder: Select Inbox (or whichever folder to monitor)
  2. Include Attachments: Set to Yes
  3. Only with Attachments: Set to No (we want all emails, not just ones with attachments)
  4. Leave all other fields at their defaults

Step 3: Add "Html to text" Conversion

The email trigger only provides the body in HTML format. We need plain text for the app to parse reliably.

  1. Click + New step
  2. Search for "Html to text" (it's under "Content Conversion")
  3. In the Content field, click inside it and select the dynamic value: Body (from the email trigger)
Why not use bodyPreview? The bodyPreview field is truncated to 255 characters, which is too short for most emails. The Html-to-text conversion gives us the full plain text body.

Step 4: Compose the Metadata JSON

This step builds a JSON file with all the email details the app needs.

  1. Click + New step
  2. Search for "Compose" (under Data Operations)
  3. In the Inputs field, build the following JSON. Use the Dynamic content panel to insert the blue tokens for each field:
{
  "from_address": "[Dynamic: From]",
  "from_name": "[Dynamic: From]",
  "subject": "[Dynamic: Subject]",
  "received_at": "[Dynamic: Received Time]",
  "body_text": "[Dynamic: output from Html to text step]",
  "body_html": "[Dynamic: Body]"
}
Tip: Click inside the Inputs field, type the opening {, then type each key (like "from_address": ) and click the corresponding blue dynamic content token to insert the value. The tokens will appear as blue pills in the field.

Step 5: Save metadata.json to OneDrive

This creates the metadata file in a structured folder in OneDrive.

  1. Click + New step
  2. Search for "Create file" (OneDrive for Business)
  3. Folder Path: Click the folder icon and type the following expression. You'll need to use the Expression tab for the formatDateTime parts: 
    /TrailerPark/@{formatDateTime(triggerOutputs()?['body/receivedDateTime'], 'yyyy-MM')}/@{formatDateTime(triggerOutputs()?['body/receivedDateTime'], 'yyyy-MM-ddTHH-mm-ss')}

    This creates paths like: /TrailerPark/2026-05/2026-05-09T08-30-00/

  4. File Name: metadata.json
  5. File Content: Select the Output from the Compose step (Step 4)
How folder paths work: The monthly folder (e.g., 2026-05) groups emails by month. The timestamp folder (e.g., 2026-05-09T08-30-00) is unique per email. Folders are created automatically when the file is saved — you don't need to create them manually. We use timestamp-only names (no email subject) to avoid issues with special characters in file paths.

Step 6: Save Attachments (Skip Signature Images)

  1. Click + New step
  2. Search for "Apply to each" (under Control)
  3. In "Select an output from previous steps", choose Attachments (from the email trigger)
  4. Inside the Apply to each loop, click Add an action → search for "Condition" (Control)
  5. Set the condition:
    • Left side: click Dynamic content → select Is Inline (from the attachment)
    • Operator: is equal to
    • Right side: type false
  6. In the "If yes" branch (attachment is NOT inline), add "Create file" (OneDrive for Business):
    • Folder Path: Same expression as Step 5: 
      /TrailerPark/@{formatDateTime(triggerOutputs()?['body/receivedDateTime'], 'yyyy-MM')}/@{formatDateTime(triggerOutputs()?['body/receivedDateTime'], 'yyyy-MM-ddTHH-mm-ss')}
    • File Name: Select Name (from the current attachment)
    • File Content: Select Content Bytes (from the current attachment)
  7. Leave the "If no" branch empty (this skips inline signature images)

Step 7: Save and Test

  1. Click Save in the top-right corner
  2. Send a test email to the inbox (include a PDF or image attachment for a full test)
  3. Wait 1-2 minutes for the flow to trigger
  4. Check OneDrive (online at onedrive.com or in File Explorer): you should see a new folder at TrailerPark/2026-05/2026-05-09Txx-xx-xx/
  5. Open the folder — verify metadata.json is there and contains the email details
  6. Verify any attachments are saved alongside it

Step 8: Confirm Local Sync

Make sure the OneDrive desktop app is syncing the TrailerPark folder to the office computer.

  1. On the office computer, click the OneDrive icon in the system tray (bottom-right)
  2. Go to Settings → Account → Choose folders
  3. Make sure the TrailerPark folder is checked for sync
  4. Note the local path where it syncs to (usually something like): 
    C:\Users\YourName\OneDrive - Truth Truck Group\TrailerPark
  5. This path is what goes into the TrailerPark app's settings as EMAIL_DIR

Troubleshooting

ProblemSolution
Flow doesn't trigger on new emails Go to My flows → click the flow → check Run history. Make sure the trigger folder matches your inbox. Try turning the flow off and back on.
Folder not appearing on local computer Check OneDrive sync status (cloud icon in system tray). Make sure the TrailerPark folder is selected for sync. Right-click the OneDrive icon and choose "Sync now".
Two emails arrive at the exact same second Rare at 40-100 emails/day. If it happens, the second folder creation will fail. Fix by adding _@{rand(1000,9999)} to the end of the folder name expression in Steps 5 and 6.
Large attachments fail to save OneDrive has a 100 MB file size limit per action. Attachments larger than this will fail. This is extremely rare for email attachments.
Flow shows errors in run history Click the failed run to see which step failed and the error message. Common issues: expired M365 session (re-authenticate the connections), OneDrive storage full.
Important: The TrailerPark folder must stay synced on the office computer for the app to work. If OneDrive sync is paused or the folder is unselected, the app won't see new emails.

Once It's Working

After confirming the flow works and files are syncing locally:

  1. Let it run for a day to accumulate some real email data
  2. Note the local folder path (the EMAIL_DIR for the app)
  3. The TrailerPark app will be configured to scan this folder and process the emails automatically
